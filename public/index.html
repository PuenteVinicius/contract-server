Overview

Introdução

A API de mock server ou contract server, é uma simples API onde é possível mockar qualquer tipo de retorno desejado a partir de um contrato JSON e de um verbo HTTP. Essa API aceita como argumento arquivos JSON que devem ter uma estrutura especifica (ver seção tal) que será explicada nas seguintes seções.

Essa API foi criada com um intuíto de se agilizar o trabalho de desenvolvimento do front-end em conjunto com o back-end, a principal ideia é que ambos profissionais definam um contrato e seus possíveis retornos, para que o front-end consiga livremente alterar o endpoint da forma que deseja sem necessitar de pausar o backend em suas atividades.

Funcionamento da API

Para criar os retornos desejados todos os arquivos JSON deverão ser criados na pasta contracts na raiz do projeto, você pode criar ela manualmente, ou simplismente rodar o comando $ npm start que o diretório será criado automaticamente, seguindo a estrutura padrão que será vista na próxima seção será possível escolher o retorno, da rquisição assim como o seu body, e como seu verbo HTTP. Os verbos suportados até atual versão (v.1.0.0) são: GET, POST, PUT e DELETE.

Referência

Primeiros Passos

Vamos supor que se é necessário criar um endoint que retorna uma lista de placa de veículos, então o arquivo placas.JSON que será criado no diretorio /contracts (pode possuir qualquer nome) deverá seguir a seguinte estrutura: 


{ 
  "path": "/placas", 
  "method": "GET", 
  "response": [
    "JVT-4203",
    "NFA-4120",
    "AZG-2279",
    "HOS-8411",
    "NEK-7930"
  ] 
}

Todo JSON terá os mesmos atributos path, method, response e no caso do dos metódos PUT e POST o atributo body segue a baixo uma lista do que cada atributo serve:

  • path: rota em que request será feita.

  • method: método HTTP que será utilizado podenso ser (POST, GET, PUT, DELETE)

  • response: a reposta esperada contrato.

  • body: no caso dos métodos PUT e POST se refere ao body da requisição.

  • options(opcional): nesse atributo pode-se colocar o que quiser, diferentes estados para registro.

POST

Agora vamos supor que se é necessário criar um endoint que cadastra um cliente:
clientes.json 

{
  "path": "/clientes",
  "method": "POST",
  "response": "success",
  "body": {
    "nome": "Flávia Priscila Araújo",
    "cpf": "390.177.376-20",
    "rg": "38.821.892-7",
    "sexo": "F",
    "mae": "Maya Sophie",
    "pai": "Bernardo Gael Márcio Araújo"
  }
}

Agora é possível fazer uma requisição POST para o /clientes de acordo com o que foi definido no body, caso envie algo diferente do contrato, será retornado um erro.

PUT

Agora vamos ao mesmo exemplo do POST utilizando PUT.

  "method": "PUT",

Agora é possível fazer uma requisição PUT para o /clientes de acordo com o que foi definido no body, caso envie algo diferente do contrato, será retornado um erro.

DELETE

Agora com um exemplo utilizando DELETE.
client-delete.json 


{ 
  "path": "/cliente/c788023b-8d0e-4197-a288-6fc442d1a7b9", 
  "method": "DELETE", 
  "response": "success"
}

Agora é possível fazer uma requisição DELETE para o /cliente/c788023b-8d0e-4197-a288-6fc442d1a7b9 e irá retorunar uma mensagem de sucesso
